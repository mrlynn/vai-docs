# Roadmap: v1.31.0 Nano Docs and Release Alignment

## Goal

Ship a complete `v1.31.0` documentation update for `vai` that makes `voyage-4-nano` local inference a first-class, discoverable, and well-explained capability across the site.

## Success Criteria

- All visible "current release" messaging is aligned to `v1.31.0`
- Nano/local inference is documented in the homepage, installation flow, model docs, command docs, and dedicated deep-dive pages
- The Python bridge is explained clearly as part of the local inference story
- The docs communicate the relationship between nano and the rest of the Voyage 4 family
- A `v1.31.0` release-note blog post exists and links users to key new docs
- Navigation supports discovery of the new local inference content

## Phase Breakdown

### Phase 1: Discovery and Planning

Purpose:

- capture current state
- confirm scope and decisions
- define execution order

Outputs:

- Memory Bank foundation
- `.planning` research and roadmap artifacts
- executable phase plan

### Phase 2: Entry-Point and Release Messaging

Purpose:

- make the current release visible and accurate at first touch

Target surfaces:

- `docusaurus.config.ts` announcement bar
- `docs/intro.md`
- `docs/getting-started/installation.md`
- any other "current release" messaging surfaces found during implementation

### Phase 3: Reference Updates

Purpose:

- update core reference pages so local inference is not treated as an afterthought

Target surfaces:

- `docs/models/voyage-4-family.md`
- `docs/commands/embeddings/embed.mdx`
- `docs/commands/rag-pipeline/pipeline.md`
- `docs/commands/tools-and-learning/explain.mdx`

### Phase 4: New Nano and Local Inference Pages

Purpose:

- provide comprehensive documentation beyond short mentions on reference pages

Expected outputs:

- setup and usage guide
- local inference conceptual overview
- sidebar integration

### Phase 5: Release Notes and Final Normalization

Purpose:

- publish the launch story and catch remaining stale release references

Expected outputs:

- new `v1.31.0` blog post
- normalized version references across live docs surfaces
- final consistency pass

### Phase 6: Validation and Documentation State Update

Purpose:

- ensure docs integrity and preserve context for future work

Expected outputs:

- local build or typecheck validation
- Memory Bank progress update
- final implementation summary

## Dependencies

- `vai-nano-local-inference-spec.md` remains the primary feature source
- new docs pages may require `sidebars.ts` changes
- build validation depends on all new docs paths and links being correct

## Non-Goals

- changing the CLI implementation
- introducing Docusaurus doc versioning
- rewriting unrelated historical blog content beyond release normalization needs
